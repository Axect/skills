# Build hygiene

## The cardinal rule

**NEVER run `pdflatex`, `bibtex`, or any TeX command inside the Overleaf sync folder.**

The sync folder is mirrored to Overleaf via the `overleap` skill. Build artefacts (`.aux`, `.log`, `.pdf`, `.bbl`, `.toc`, `.out`) created inside the sync folder will be pushed to Overleaf on the next sync, polluting the remote project and potentially overwriting Overleaf-side artefacts.

If you see `<PROJECT>/main.aux`, `<PROJECT>/main.log`, or similar inside the sync folder, treat it as an emergency:

```bash
cd ~/zbin/OverLeaf/<PROJECT>
rm -f main.aux main.bbl main.blg main.log main.out main.pdf main.toc
ls main.*   # should show only main.tex
```

Then check if the user accidentally compiled there, or if a build script has a misconfigured `cd`.

## Out-of-tree build pattern

All builds run in `<PROJECT>_build/`, with source files in `<PROJECT>/`:

```
~/zbin/OverLeaf/
├── <PROJECT>/              # sync folder (Overleaf)
│   ├── main.tex            # source
│   ├── ref.bib             # bibliography
│   ├── JHEP.bst            # bibstyle (read-only, mirrored from journal)
│   ├── jcappub.sty         # class (read-only)
│   └── figs/               # figures (PDFs)
├── <PROJECT>_build/        # build artefacts only
│   ├── build_main.sh       # build script
│   ├── main.aux            # generated by pdflatex
│   ├── main.pdf            # final output
│   └── ...
└── <PROJECT>_draft/        # Korean drafts (markdown)
```

`pdflatex` is invoked from `SRC_DIR` (so `\includegraphics{figs/...}` resolves) with `-output-directory=OUT_DIR` (so all artefacts land outside the sync folder).

`.bst` / `.sty` / `ref.bib` are *copied* (not symlinked) to `OUT_DIR` before bibtex runs, because bibtex looks in the current directory.

## Build script template

See `templates/build_template.sh`. Copy it into `<PROJECT>_build/` and edit only the `JOBNAME` variable:

```bash
#!/usr/bin/env bash
# Build script for OSPREY JCAP paper.
# Compiles ../OSPREY/JCAP/<JOBNAME>.tex into this directory.
# Runs: pdflatex -> bibtex -> pdflatex -> pdflatex (standard 4-pass cycle).

set -euo pipefail

SRC_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")/../OSPREY/JCAP" && pwd)"
OUT_DIR="$(cd "$(dirname "${BASH_SOURCE[0]}")" && pwd)"
JOBNAME="main"   # <-- edit this per .tex file

cd "$SRC_DIR"
cp -f JHEP.bst jcappub.sty ref.bib "$OUT_DIR/"

PDFLATEX_ARGS=(-interaction=nonstopmode -halt-on-error -output-directory="$OUT_DIR")

run_pdflatex() {
    local label="$1"
    echo ">>> pdflatex pass: $label"
    if ! pdflatex "${PDFLATEX_ARGS[@]}" "${JOBNAME}.tex" > "$OUT_DIR/${JOBNAME}.${label}.log" 2>&1; then
        echo "!!! pdflatex failed on pass '$label'. Tail of log:"
        tail -40 "$OUT_DIR/${JOBNAME}.${label}.log"
        exit 1
    fi
}

run_pdflatex "1"

echo ">>> bibtex"
( cd "$OUT_DIR" && bibtex "$JOBNAME" > "${JOBNAME}.bibtex.log" 2>&1 ) || {
    echo "!!! bibtex failed. Tail of log:"
    tail -40 "$OUT_DIR/${JOBNAME}.bibtex.log"
    exit 1
}

run_pdflatex "2"
run_pdflatex "3"

echo
echo "Build complete: $OUT_DIR/${JOBNAME}.pdf"
ls -la "$OUT_DIR/${JOBNAME}.pdf"

echo
echo "--- Warnings from final pass ---"
grep -E "(undefined|Overfull|Underfull|Error)" "$OUT_DIR/${JOBNAME}.3.log" | head -30 || true
```

One script per major `.tex` file (e.g., `build_main.sh` for `main.tex`, `build_note.sh` for `note.tex` if both coexist).

## Common build failures and fixes

### Failure: `\toprule` undefined

```
! Undefined control sequence.
l.348   \toprule
```

Cause: `booktabs` package not loaded. Tables using `\toprule`/`\midrule`/`\bottomrule` need it.

Fix: add to preamble of `main.tex`:
```latex
\usepackage{booktabs}
```

### Failure: All `\cite{}` undefined despite ref.bib existing

```
Package natbib Warning: Citation `Hawking:1975vcx' on page 4 undefined ...
```

Cause: `main.tex` uses placeholder `\begin{thebibliography}{99}` ... `\end{thebibliography}` instead of `\bibliography{ref}`.

Fix: replace the manual `thebibliography` block with:
```latex
\bibliographystyle{JHEP}
\bibliography{ref}
```

### Failure: `pdflatex: I can't find file 'figs/foo.pdf'`

Cause: PDF was not copied from `outputs/<topic>/plots/` to `<PROJECT>/figs/`.

Fix:
```bash
cp ~/Documents/Project/Research/<PROJECT>/operator/outputs/<topic>/plots/foo.pdf \
   ~/zbin/OverLeaf/<PROJECT>/figs/
```

### Failure: Specific equation `\ref{eq:foo}` undefined despite being defined

Cause: First pdflatex pass not yet complete; build hadn't run pass 2.

Fix: build script runs 3 pdflatex passes for this exact reason. If still undefined after pass 3, the label is genuinely missing or misspelled.

## Build acceptance criteria

A build is "clean" if:

1. `bash build_<jobname>.sh` exits 0.
2. `<JOBNAME>.pdf` exists in `<PROJECT>_build/`.
3. `grep -ci undefined <PROJECT>_build/<JOBNAME>.3.log` returns `0`.

Overfull / Underfull hbox/vbox warnings are cosmetic and do NOT block acceptance. They can be addressed pre-submission.

## Build cleanup

When abandoning a draft or switching jobs:

```bash
cd ~/zbin/OverLeaf/<PROJECT>_build/
rm -f *.aux *.bbl *.blg *.log *.out *.pdf *.toc *.bibtex.log *.1.log *.2.log *.3.log
# keep build_*.sh, ref.bib (copy of), JHEP.bst (copy of), jcappub.sty (copy of)
```

The build script is idempotent — re-running cleans up partial state automatically.

## Overleaf sync interaction

The user runs `overleap pull` / `overleap push` manually. The skill should:

- NOT push or pull on its own (changes the user's working state).
- Treat `<PROJECT>/` as read-write for source edits, but track that anything dropped there will sync.
- Never commit `.aux`/`.log`/`.pdf` to the sync folder.

If the user asks "왜 main.pdf 가 안 보여?", the answer is `<PROJECT>_build/main.pdf` — the build does not produce a PDF in the sync folder by design.
